\chapter*{Abstract}
This Bachelor’s Final Project presents the design and development of \textit{Sevillaneando}, a cross-platform mobile application aimed at centralising the cultural and leisure event offer in Seville, with special emphasis on giving visibility to emerging and small-scale events that go unnoticed through existing platforms.

The system addresses a real information fragmentation problem: relevant events are scattered across social networks, institutional websites and word of mouth, making it difficult for citizens to discover what is happening near them. \textit{Sevillaneando} solves this by combining a participatory model — where any registered user can publish and manage their own events — with an automatic scraping engine that imports events from external sources such as Eventbrite and Ticketmaster.

The application provides a full feature set: event listing with filtering by category, price, date and geographic radius; an interactive map view; public and private events with a moderation workflow; a personalised recommendation engine based on user behaviour and interests; route planning with three optimisation strategies; real-time group and private chat via WebSockets; social features (following, reviews, notifications); and role-based access control for users, moderators and administrators.

From a technical standpoint, the project integrates React Native and Expo for the mobile frontend, NestJS and Node.js for the REST API and WebSocket server, PostgreSQL 15 with the PostGIS extension for geospatial queries, Firebase Authentication for identity management, and Cloudinary for image storage. The backend is deployed on Google Cloud Run.
